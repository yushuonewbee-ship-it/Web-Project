# 启动说明

## 启动 GIS 地图功能

GIS 地图功能需要同时运行**前端**和**后端**两个服务器。

### 方法 1：分别启动（推荐用于调试）

**终端 1 - 启动后端服务器：**
```bash
cd Kimi_Agent_Web/app
npm run server
```
后端服务器将在 `http://localhost:4000` 运行

**终端 2 - 启动前端开发服务器：**
```bash
cd Kimi_Agent_Web/app
npm run dev
```
前端服务器将在 `http://localhost:5173` 运行（或 Vite 默认端口）

### 方法 2：使用 dev:full 脚本（Windows 可能不支持）

```bash
cd Kimi_Agent_Web/app
npm run dev:full
```

## 验证服务器是否运行

1. **测试后端 API：**
   - 在浏览器中访问：`http://localhost:4000/api/gis/test`
   - 应该看到 JSON 响应：`{"message":"GIS API 工作正常","timestamp":"..."}`

2. **测试 shp 文件 API：**
   - 在浏览器中访问：`http://localhost:4000/api/gis/shapefile`
   - 应该看到 GeoJSON 数据

## 常见问题

### 问题 1：端口被占用
如果 4000 端口被占用，可以修改 `server.mjs` 中的端口号：
```javascript
const PORT = process.env.PORT || 4001; // 改为其他端口
```
同时需要修改 `vite.config.ts` 中的代理目标：
```typescript
target: "http://localhost:4001", // 改为对应的端口
```

### 问题 2：shp 文件找不到
确保 `十八省底图.shp` 文件在 `Kimi_Agent_Web/app/` 目录下。

### 问题 3：CORS 错误
如果遇到 CORS 错误，检查 `server.mjs` 中是否已启用 CORS：
```javascript
app.use(cors());
```

## 文件要求

shp 文件及其配套文件应位于项目根目录：
- `十八省底图.shp`（必需）
- `十八省底图.shx`（推荐，用于索引）
- `十八省底图.dbf`（推荐，用于属性数据）
