# 明清农业数据库 Web 项目 — 部署说明

本项目为 **前端（Vite + React）+ 后端（Node + Express）+ MySQL** 结构。推荐使用 **Vercel（前端）+ Render（后端）** 免费部署，数据库可使用云 MySQL 或 Railway 提供的 MySQL。

---

## 一、部署前准备

### 1. 代码已支持

- **前端**：通过环境变量 `VITE_API_URL` 指定后端地址，未设置时本地走 Vite 代理。
- **后端**：通过环境变量 `MYSQL_*`、`PORT` 配置数据库与端口，见 `.env.example`。

### 2. 需要准备

- **GitHub 账号**（用于 Vercel、Render 关联仓库）
- **MySQL 数据库**（任选其一）：
  - [Railway](https://railway.app) 创建 MySQL 服务（有免费额度）
  - [TiDB Cloud](https://tidbcloud.com) 免费 MySQL 兼容库
  - 或已有云服务器 / 本机 MySQL

### 3. 仓库结构

部署时以 **Kimi_Agent_Web/app** 为项目根目录（即本目录），前端构建、后端入口都在此目录下。

- 若整仓只有这一个应用，可在 GitHub 上单独建一个仓库，只放 `Kimi_Agent_Web/app` 下的内容再部署。
- 若保留现有大仓，Vercel/Render 需将「项目根目录」或「Build 根目录」设置为 `Kimi_Agent_Web/app`（若平台支持）。

---

## 二、部署后端（Render）

1. 打开 [Render](https://render.com)，用 GitHub 登录。
2. **New → Web Service**，连接你的 GitHub 仓库。
3. **Root Directory**（若可用）：填 `Kimi_Agent_Web/app`；否则先把该目录单独建仓再连。
4. 配置：
   - **Build Command**：`npm install`
   - **Start Command**：`node server.mjs`
   - **Environment**：添加变量  
     `MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_DATABASE`、`MYSQL_USER`、`MYSQL_PASSWORD`（与云 MySQL 一致），以及 `PORT`（一般由 Render 自动提供）。
5. 若使用 **Shapefile（十八省底图）**：确保仓库中包含 `十八省底图.shp` 等文件，且部署时当前工作目录下能访问到（即与 `server.mjs` 同层级或通过路径能读到）。
6. 部署完成后记下 **服务地址**，例如：`https://your-app-name.onrender.com`。

---

## 三、部署前端（Vercel）

1. 打开 [Vercel](https://vercel.com)，用 GitHub 登录。
2. **Add New → Project**，导入同一仓库。
3. **Root Directory**：若仓库仅为 app，选根目录；否则设为 `Kimi_Agent_Web/app`（若 Vercel 支持子目录）。
4. 配置：
   - **Framework Preset**：Vite
   - **Build Command**：`npm run build`
   - **Output Directory**：`dist`
   - **Environment Variables**：添加  
     `VITE_API_URL` = 上一步得到的后端地址（如 `https://your-app-name.onrender.com`），**不要**在末尾加 `/`。
5. 部署后即可通过 Vercel 提供的域名访问前端；前端会请求 `VITE_API_URL` 下的 `/api/*`。

---

## 四、本地与生产环境对照

| 环境     | 前端 API 请求方式              | 后端数据库来源     |
|----------|---------------------------------|--------------------|
| 本地开发 | `VITE_API_URL` 不设，走 Vite 代理 | 本机 MySQL 或 .env |
| 生产     | `VITE_API_URL` = Render 后端地址  | 环境变量 MYSQL_*   |

---

## 五、常见问题

- **数据库连接失败**：检查云 MySQL 是否允许外网访问、白名单是否包含 Render 出口 IP；`MYSQL_DATABASE` 名称若含空格，需与库名完全一致。
- **GIS 无数据**：确认后端仓库内包含 `十八省底图.shp`（及配套 .dbf 等），且 `server.mjs` 中读取路径与部署后的工作目录一致。
- **跨域错误**：后端已启用 `cors()`，一般无需改；若仍有问题，可在后端为 `VITE_*` 前端域名单独放行。

部署完成后，将 Vercel 域名分享即可供他人访问前端；数据库与后端由你在 Render 与 MySQL 配置中维护。
